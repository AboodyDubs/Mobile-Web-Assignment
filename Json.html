1- What is Json?

Javascript Object Notation is a specific file format that uses simple text that can be read by the ordinary human eye. It can store data through the systematic usage of Javascript statements in an html code. It is important to note that a Json contains a file on its own but relies on being called from an HTML file. thus it must be called from the html file in order to function properly.



2- Characteristics of Json:

  1. Human-readable format: can be understood by humans (data is not in a binary format)

  2. Compatible with HTML: can be used with operations created through HTML tags

  3. Javascript-compatible: can be made interactive through specific statements written in the script block

  4. Size: it is genrally a small sized file that adds up to a couple of megabytes in total.

  5. Independent: does not completely rely on other languages such as html to work

3- History of Json

JSON existed since the early 1990s. However, it was not paid much attention to at the time. A computer programmer who goes by the name Douglas Crockford was the first person to explain how to use the JSON format in 2001. He was credited as the person who defined what JSON was. Thus the name was given by him as he understood that it was transporting javascript data.

JSON was at first understood to be the correct format in delivering javascript objects
into web browsers, whereas we have different understanding of it now because it transports
the format with any language not just javascript.

4- Comparing JSON with XML


There are multiple differences between Json and XML:


  1. Array: Json can function with arrays, where as XML cannot.

  2. Readibility and comprehension: XML is less understandable than JSON.

  3. Datatype compatibility: JSON cannot contain datatypes such as images or videos or audio.

  4. JSON is not defined as a language, unlike XML.
